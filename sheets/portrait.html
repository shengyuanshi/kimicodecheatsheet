pnpm install Alternative package manager pnpm add -g @moonshot-ai/kimi-code

Fast Start

kimi in a repo. Use /init once, /plan for risky work, and /compact when context gets long.

Trust Rule

--yolo is for trusted workspaces only. Keep approvals on when touching unfamiliar code or production data.

Start Fast

Open or resume

kimiOpen Kimi Code in the current repo.
kimi --continueResume the latest session here.
kimi --sessionPick an older session to resume.

Run from shell

kimi -p "..."Run one non-interactive prompt.
--output-format stream-jsonJSONL output for automation.
--model <model>Start with a specific model.

Modes

kimi --planStart with Plan Mode on.
kimi --yoloAuto-approve routine actions.
kimi --autoAuto-handle approvals without asking.
--skills-dir <dir>Load custom Skills.
CLI Automation

Non-interactive prompts

kimi -p "..."Run one prompt and print the response.
kimi --prompt "..."Long-form equivalent of -p.

Structured output

--output-format stream-jsonEmit JSONL events for scripts.
--model <model>Select a model for the run.

Session targeting

--continueResume latest session in this directory.
--session <id>Resume a specific session.
kimi export [id]Export a session bundle.

Plan Mode
Use Plan Mode when the task is broad, risky, unclear, or likely to touch multiple files.

Good for

Large refactors
Architecture changes
Migration planning
Bug investigation
PR review before edit
Ambiguous requests

Controls

Shift TabToggle Plan Mode
/plan onEnter Plan Mode
/plan offExit Plan Mode
/plan clearClear current plan
EnterPlanModeTool enters Plan Mode.
ExitPlanModeTool exits after plan approval.
Permissions
Low risk: reading files, searching code, checking status.
Needs care: file writes, edits, shell commands, and task stops.
Trusted repo: /yolo or kimi --yolo can reduce approval friction.
Unknown repo: keep approvals on until you understand the blast radius.
